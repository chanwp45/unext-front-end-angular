# Commit Message Convention

## Format

```
<type>(<scope>): <subject>

<body>

<footer>
```

## Commit Types

| Type | Description |
|------|-------------|
| `feat` | A new feature |
| `fix` | A bug fix |
| `docs` | Documentation only changes |
| `style` | Changes that don't affect code meaning |
| `refactor` | Code change without feature/bug change |
| `perf` | Code change for performance |
| `test` | Adding missing tests |
| `chore` | Changes to build process, dependencies |

## Examples

```
feat(auth): add two-factor authentication
fix(login): resolve password validation bug
docs(readme): update installation steps
refactor(components): simplify button component
perf(api): optimize data fetching
test(user-profile): add unit tests
```
