# Team Guidelines

## Mandatory Requirements (ต้องปฏิบัติ)

- All code must follow **Naming Conventions** → [Coding Standards](./coding-standards.md)
- All commits must follow **Commit Message Convention** → [Commit Convention](./commit-convention.md)
- All components must have **unit tests** (80% coverage minimum) → [Unit Test Standard](./unit-test-standard.md)
- All PRs must pass **Definition of Done** checklist → [Definition of Done](./definition-of-done.md)
- Code must pass **ESLint + Prettier** validation
- TypeScript must have **strict mode** enabled

## For New Projects

1. Copy [CLAUDE.md](../../CLAUDE.md) to your project root
2. Replace all `[PLACEHOLDER_NAME]` values with actual project details
3. Commit to version control
4. Reference in `README.md` for team onboarding

## For Existing Projects

1. Create or update `CLAUDE.md` in project root
2. Review current standards against SOP section
3. Document any deviations with explanations
4. Share with team for alignment and feedback
5. Update regularly as project evolves
